变量声明

let关键字

rust是强类型语言，使用let关键字声明变量，会自推导类型，也可以指定类型。

可变变量、不可变变量、常量

  • 可变变量允许直接重新赋值。
  • 不可变变量，只能重影（shadowing），类似一种重新绑定。
  • 常量不可变。

基本数据类型

注释

严格来说不属于数据类型。rust的注释和C的相同。

整型

位长度 有符号 无符号
8-bit i8 u8
16-bit i16 u16
32-bit i32 u32
64-bit i64 u64
128-bit i128 u128
架构 isize usize
进制 例
十进制 12_345 或 12345
十六进制 0xff
八进制 0o77
二进制 0b1111_0000（推荐）
字节(只能表示 u8 型) b’A’

Tips：

下划线的设计能方便程员计数，当然添加下划线也是可以的。

浮点

有两种类型：

  • f32：32位浮点
  • f64：64位浮点

布尔

使用bool关键字进行声明，有两个取值。

  • ture
  • false

Tips：

在判断中，rust严格使用布尔类型，不同于其他语言的非零即真。

字符

使用char关键字进行声明，rust的字符型占4字节，是Unicode，使用单引号。

个人常用的迭代器方法

next

返回迭代器中的下一个元素

count

计算迭代器中的元素数量。

nth

返回迭代器中第 n 个元素。

last

返回迭代器中的最后一个元素。

all

如果迭代器中的所有元素都满足某个条件，返回true。

any

如果迭代器中的至少一个元素满足某个条件，返回true。

find

返回迭代器中第一个满足某个条件的元素，是一个Option枚举对象。

map

对迭代器中的每个元素应用一个函数。

filter

保留迭代器中满足某个条件的元素。

take_while

从迭代器中取出满足某个条件的元素，直到不满足为止。

skip_while

跳过迭代器中满足某个条件的元素，直到不满足为止。

for_each

对迭代器中的每个元素执行某种操作。

fold

对迭代器中的元素进行折叠，使用一个累加器。

take

从迭代器中取出最多 n 个元素。

zip

将两个迭代器中的元素打包成元组。

rev

反转迭代器中的元素顺序。

enumerate

为迭代器中的每个元素添加索引，即将原来每一个元素变为一个元组对象(索引, 元素)

step_by

按照指定的步长返回迭代器中的元素。

inspect

在取出每个元素时执行一个闭包，但不改变元素。

数组不同于元组，数组要求使用相同的数据类型。

这种访问方式存在越界的风险，越界之后终止运行进入backtrace

不推荐，一般用于可变数组。

分割

split

split_whitespace

按空格切分字符串，和python的split函数default的功能类似。

大小写转换

去除首尾字符

枚举

普通枚举类型

rust的枚举类型比较复杂，可以在枚举类型中定义数据类型。使用enum定义一个枚举。

rust的枚举可以定义包含的类型（每个枚举项的类型可以不同，每个枚举项可以有多个类型）

Tips:

在枚举对象前添加#[derive(Debug)]之后可以进行打印

Option枚举类型

rust没有NULL类型，引入Option枚举来解决这个问题（限制NULL的声明以提高安全性）。

如上可知，Option枚举是一个泛型，所以初始值为空的 Option 必须明确类型

由于Option是编译器默认引入，故可省略不写
